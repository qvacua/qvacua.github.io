Typography for Developers Using Markdown

At UX Munich Marko Dugonjić (@markodugonjic) gave a very nice talk "Typography for Developers"; find the slides here. In his talk he demonstrated in a step by step manner, how easy it is—even for developers—to apply beautiful and typography using not very complicated CSS. His demo CSS is indeed very nice and simple.

I basically used his demo CSS in the first version of LookDown. However, I realized very soon the "demo" part of his CSS: It is not complete. Some examples include:

  • Nested lists do not work,
  • Long headings do not respect the vertical rhythm.

Thus, I needed to improve it, especially geared towards the rendering of Markdown documents, and the following is the result of few hours of work. It still is based on Marko's CSS, but with the following enhancements:

  • Nested lists work,
  • Headings from level 1 to 6,
  • Long headings, ie those spanning multiple lines, respect the vertical rhythm,
  • Heveltica Neue for the headings: Just my personal taste,
  • Indented blockquote,
  • Used rem instead of em to avoid nesting issues,
  • hr respects the vertical rhythm.

For the source to https://github.com/qvacua/typo. It may well be that I totally misunderstood the vertical rhythm thing and proper sizing of things, but I like it and that's all that counts. =)
